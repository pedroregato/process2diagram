Google Calendar — Setup inicial

⚠️ Execute os dois scripts SQL de migração no Supabase antes de usar qualquer funcionalidade de calendário. Veja a seção SQL Migrations.

1. Service Account no Google Cloud

  1. Acesse console.cloud.google.com → IAM & Admin → Service Accounts
  2. Crie ou use a conta p2d-calendar-sa@process2diagram.iam.gserviceaccount.com
  3. Aba Chaves → Adicionar chave → Criar nova chave → JSON
  4. Baixe o arquivo .json gerado
  5. Verifique se a Google Calendar API está habilitada no projeto Cloud

2. Configurar Secrets no Streamlit Cloud

[google_calendar]
calendar_id      = "xxx@group.calendar.google.com"
credentials_json = '''COLE_AQUI_TODO_O_CONTEÚDO_DO_JSON_BAIXADO'''
⚠️ Use ''' (três aspas simples) para envolver o JSON — nunca """. Aspas duplas processam \n e corrompem a chave privada.

3. Compartilhar a agenda com a Service Account

  1. Abra calendar.google.com
  2. Localize a agenda → três pontos → Configurações e compartilhamento
  3. Adicione p2d-calendar-sa@process2diagram.iam.gserviceaccount.com
  4. Permissão: "Fazer alterações e gerenciar compartilhamento" (necessário para as ferramentas de ACL)

4. Verificar com o diagnóstico

No Assistente P2D (qualquer perfil), diga:

Execute o diagnóstico do Google Calendar

Resultado esperado: todos os 7 passos ✅ (incluindo 2b RSA e 4b token OAuth).

Google Calendar — Multi-projeto

Cada projeto pode ter seu próprio calendar_id armazenado no Supabase. Quando o Assistente opera no contexto de um projeto, a agenda específica daquele projeto é usada automaticamente. Se não houver configuração, usa o calendar_id global dos st.secrets.

Como configurar um calendar_id por projeto
  1. Acesse Configurações → aba Banco de Dados
  2. Role até a seção 📅 Google Calendar por Projeto (visível apenas para admin)
  3. Clique em ➕ Associar / Atualizar calendar_id
  4. Selecione o projeto e cole o Calendar ID da agenda desejada
  5. Clique em 💾 Salvar
💡 O Calendar ID está em Google Calendar → Configurações da agenda → Integrar agenda → "ID da agenda" (formato: xxx@group.calendar.google.com).

Compartilhamento de agenda

As ferramentas calendar_share_with_user e calendar_revoke_access usam a Google Calendar ACL API para gerenciar permissões diretamente pelo chat do Assistente.

⚠️ A Service Account precisa ter permissão "Fazer alterações e gerenciar compartilhamento" na agenda (não apenas "Fazer alterações nos eventos"). Sem isso, a ferramenta retorna erro 403.
RoleO que permite
readerApenas visualizar eventos
writerCriar, editar e excluir eventos (padrão)
ownerGerenciar agenda e compartilhamento

  2. Selecione o domínio
  3. Na seção Usuários, expanda o usuário desejado
  4. Role até a linha Contas de integração
  5. Preencha Conta Google e/ou Conta Microsoft Teams
  6. Clique em 💾 para salvar

SQL Migrations — scripts obrigatórios

⚠️ Execute estes scripts no SQL Editor do Supabase antes de usar as funcionalidades. Cada script é idempotente (IF NOT EXISTS / ADD COLUMN IF NOT EXISTS).

1. Contas de integração por usuário

Arquivo: setup/supabase_migration_user_accounts.sql

ALTER TABLE tenant_users
    ADD COLUMN IF NOT EXISTS google_account   TEXT DEFAULT NULL,
    ADD COLUMN IF NOT EXISTS ms_teams_account TEXT DEFAULT NULL;

2. Calendar ID por projeto

Arquivo: setup/supabase_migration_project_calendar.sql

CREATE TABLE IF NOT EXISTS project_calendar_config (
    project_id  UUID PRIMARY KEY REFERENCES projects(id) ON DELETE CASCADE,
    calendar_id TEXT NOT NULL,
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
ALTER TABLE project_calendar_config ENABLE ROW LEVEL SECURITY;
