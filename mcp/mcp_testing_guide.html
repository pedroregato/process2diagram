Visao Geral

O servidor MCP (Model Context Protocol) expoe operacoes do Google Calendar diretamente ao Claude Code. Ao conversar com o Claude, basta descrever o que quer fazer em linguagem natural — o Claude seleciona e executa a ferramenta adequada automaticamente.

Como funciona: o arquivo .mcp.json na raiz do projeto registra o servidor. Quando o Claude Code inicia, ele lanca o processo python mcp/google_calendar_server.py e disponibiliza as 8 ferramentas na sessao ativa.

Ferramentas disponíveis

list_events
Lista proximos eventos com filtros de data e busca por texto
get_event
Retorna detalhes completos de um evento pelo ID
create_event
Cria novo evento com titulo, horario, local e participantes
update_event
Atualiza campos de um evento existente (patch parcial)
delete_event
Remove um evento e envia notificacao aos participantes
list_calendars
Lista todas as agendas acessiveis pela Service Account
respond_to_event
Atualiza o RSVP de um participante (aceito/recusado/tentativo)
suggest_time
Sugere horarios livres usando a API freebusy do Google

Pre-requisitos

  • ✓
    Servidor MCP configurado
    .mcp.json aponta para mcp/google_calendar_server.py
  • ✓
    Service Account JSON presente
    mcp/google_console/process2diagram-*.json — ignorado pelo git
  • ✓
    Calendar ID configurado
    mcp/google_console/.google-calendar — ignorado pelo git
  • !
    Dependencias instaladas
    Execute: pip install -r mcp/requirements.txt
  • !
    Service Account com acesso ao calendario
    No Google Calendar, compartilhe a agenda com o e-mail da Service Account com permissao "Fazer alteracoes nos eventos"

Instalando as dependencias

# No terminal, dentro do diretorio do projeto
pip install -r mcp/requirements.txt

Verificando se o servidor esta ativo

No chat do Claude Code, pergunte:

Voce: Quais ferramentas MCP de calendario estao disponiveis?

O Claude deve listar as 8 ferramentas. Se nao listar, o servidor nao foi iniciado corretamente — verifique a secao Solucao de Problemas.

Arquitetura

    |
    |-- le .mcp.json
    |
    v
python mcp/google_calendar_server.py   <-- FastMCP via stdio
    |
    |-- Carrega credenciais: mcp/google_console/*.json
    |-- Le calendar ID:      mcp/google_console/.google-calendar
    |
    v
Google Calendar API v3
    |
    v
Agenda compartilhada com a Service Account
O servidor MCP roda como processo filho do Claude Code, comunicando via stdio (stdin/stdout). Nao ha porta de rede aberta. As credenciais nunca trafegam pela rede local.

Catalogo de Ferramentas

Cada ferramenta pode ser invocada diretamente no chat do Claude Code. Os exemplos abaixo mostram tanto o prompt em linguagem natural quanto o equivalente tecnico para referencia.

list_events

Lista proximos eventos da agenda. Suporta filtros de periodo e busca por texto.

GET Parametros
ParametroTipoObrigatorioDescricao
max_resultsintopcionalMaximo de eventos (padrao 10, max 250)
time_minstringopcionalInicio do periodo (ISO 8601). Padrao: agora
time_maxstringopcionalFim do periodo (ISO 8601)
querystringopcionalFiltro por texto livre no titulo/descricao
calendar_idstringopcionalID da agenda. Padrao: agenda do projeto

Exemplos de uso

Proximos 10 eventos
Voce: Quais sao os proximos eventos do calendario?
Filtrar por periodo
Voce: Liste os eventos de maio de 2026.
Busca por texto
Voce: Tem algum evento de "reuniao de sprint" agendado?
Quantidade especifica
Voce: Mostre os proximos 3 eventos do calendario.

get_event

Retorna todos os detalhes de um evento especifico pelo seu ID. O ID e obtido via list_events.

GET Parametros
ParametroTipoObrigatorioDescricao
event_idstringobrigatorioID do evento (obtido via list_events)
calendar_idstringopcionalID da agenda. Padrao: agenda do projeto

Exemplos de uso

Fluxo tipico: listar e depois detalhar
Voce: Liste os proximos eventos e me mostre os detalhes do primeiro.

-- Claude lista os eventos e chama get_event automaticamente --
Direto pelo ID
Voce: Busque os detalhes do evento com ID "abc123xyz".

create_event

Cria um novo evento no calendario. Envia convites por e-mail quando participantes sao informados.

POST Parametros
ParametroTipoObrigatorioDescricao
summarystringobrigatorioTitulo do evento
start_datetimestringobrigatorioInicio — ISO 8601 (ex: "2026-05-15T14:00:00")
end_datetimestringobrigatorioFim — ISO 8601 (ex: "2026-05-15T15:00:00")
descriptionstringopcionalDescricao ou pauta do evento
locationstringopcionalLocal ou link do Meet/Teams
attendeesstringopcionalEmails separados por virgula
calendar_idstringopcionalID da agenda. Padrao: agenda do projeto
Atencao: horarios sem fuso horario sao interpretados como America/Sao_Paulo. Exemplo: "2026-05-15T14:00:00" cria um evento as 14h no horario de Brasilia.

Exemplos de uso

Evento simples
Voce: Crie um evento "Reuniao de Sprint" amanha as 10h com duracao de 1 hora.
Com participantes e descricao
Voce: Agende uma "Revisao de Requisitos" para 20/05/2026 das 14h as 16h,
local "Sala 3 - Bloco A", com os participantes joao@empresa.com e
ana@empresa.com. Pauta: validacao dos requisitos do modulo de relatorios.
Reuniao recorrente (descricao livre)
Voce: Crie uma reuniao diaria de standup para 19/05/2026 as 09h15,
com 15 minutos de duracao. Participantes: time@empresa.com

summarystringopcionalNovo titulo
start_datetimestringopcionalNovo horario de inicio
end_datetimestringopcionalNovo horario de fim
descriptionstringopcionalNova descricao
locationstringopcionalNovo local

Exemplos de uso

Reagendar evento

no mesmo horario.
Atualizar apenas o local
Voce: Altere o local da "Reuniao de Sprint" de amanha para
"https://meet.google.com/abc-defg-hij".
Renomear evento
Voce: Renomeie o evento com ID "xyz789" para "Sprint Review — Ciclo 12".

Remove permanentemente um evento do calendario e notifica os participantes.

DELETE Parametros
ParametroTipoObrigatorioDescricao

Exemplos de uso

Direto pelo ID

list_calendars

Lista todas as agendas do Google Calendar acessiveis pela Service Account do projeto.

GET Sem parametros

Nenhum parametro necessario. Retorna id, nome, papel de acesso e flag de agenda primaria.

Exemplos de uso

Listar todas as agendas
Voce: Quais agendas a Service Account tem acesso?
Uso para obter IDs
Voce: Me mostre os IDs de todas as agendas disponiveis para eu poder usar
em outros comandos.

respond_to_event

Atualiza o status RSVP de um participante em um evento.

PATCH Parametros
ParametroTipoObrigatorioDescricao
event_idstringobrigatorioID do evento
responsestringobrigatorioaccepted | declined | tentative | needsAction
attendee_emailstringopcionalEmail do participante. Padrao: conta da Service Account
calendar_idstringopcionalID da agenda. Padrao: agenda do projeto

Exemplos de uso

Aceitar convite
Voce: Aceite o convite do evento "Reuniao de Sprint" para joao@empresa.com.
Recusar com e-mail especifico
Voce: Recuse o evento com ID "evt456" para maria@empresa.com.
Marcar como tentativo
Voce: Marque minha participacao no evento "Workshop BPMN" como tentativa.

suggest_time

Consulta a API freebusy do Google Calendar para encontrar horarios livres para os participantes informados.

POST Parametros
ParametroTipoObrigatorioDescricao
duration_minutesintopcionalDuracao necessaria em minutos (padrao 60)
attendeesstringopcionalEmails separados por virgula
time_minstringopcionalInicio da janela de busca (ISO 8601). Padrao: agora
time_maxstringopcionalFim da janela de busca. Padrao: +7 dias
max_suggestionsintopcionalNumero de sugestoes a retornar (padrao 3)
Limitacao: a API freebusy so retorna disponibilidade de calendarios que a Service Account tem acesso. Participantes externos cujos calendarios nao foram compartilhados nao sao verificados.

Exemplos de uso

Busca basica
Voce: Qual o proximo horario livre de 1 hora na proxima semana?
Com multiplos participantes
Voce: Encontre 3 opcoes de horario de 90 minutos esta semana que funcionem
para joao@empresa.com e ana@empresa.com.
Fluxo completo: sugerir e criar
Voce: Sugira um horario de 1 hora para esta semana e, se eu aprovar,
crie um evento "Alinhamento P2D" com joao@empresa.com.

Solucao de Problemas

Ferramentas nao aparecem no Claude Code

Causa mais comum: dependencias nao instaladas ou .mcp.json invalido.

# Teste o servidor manualmente:
cd D:\PythonProjects\process2diagram
python mcp/google_calendar_server.py

# Deve iniciar sem erros. Ctrl+C para sair.

Se houver ModuleNotFoundError, instale as dependencias:

pip install -r mcp/requirements.txt

Erro 403 ao chamar a API

A Service Account nao tem permissao para acessar a agenda. Verifique:

  • A agenda foi compartilhada com o e-mail da Service Account?
  • A permissao e "Fazer alteracoes nos eventos" (nao apenas leitura)?
  • A Google Calendar API esta habilitada no projeto do Google Cloud?

Erro ao carregar .google-calendar

O arquivo foi removido do git mas precisa existir localmente. Recrie-o:

# Crie o arquivo com o ID da agenda (substitua pelo ID real)
echo id-da-agenda=SEU_CALENDAR_ID@group.calendar.google.com > mcp/google_console/.google-calendar

O ID da agenda pode ser encontrado em Google Calendar → Configuracoes da agenda → "Endereco da agenda no formato iCal".

Eventos criados com horario errado

Corrigido na versao atual. Horarios sem fuso horario agora sao interpretados como America/Sao_Paulo (anteriormente eram tratados como UTC, causando desvio de 3 horas).

Erro FileNotFoundError para o JSON de credenciais

O arquivo JSON da Service Account foi removido do git mas deve existir localmente em mcp/google_console/. Faca o download novamente no Google Cloud Console:

  • IAM e Administrador → Contas de servico → Selecione a conta
  • Aba "Chaves" → Adicionar chave → JSON
  • Salve em mcp/google_console/
