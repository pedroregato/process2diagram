01 Objetivo e contexto ▶

A ValidationHub.py está referenciada no navigation group Análise de app.py mas o arquivo não existe no repo. Objetivo: criar a página consolidando em um só lugar todos os sinais de qualidade do pipeline — diagnósticos BPMN, scores do validator, quality gate da transcrição e telemetria de LLM.

Hoje esses dados estão espalhados: diagnósticos BPMN em modules/bpmn_diagnostics.py, scores em hub.validation, quality gate em ui/tabs/quality_tab.py, e telemetria em pages/LLMBenchmark.py. A ValidationHub consolida tudo para uma visão única de saúde do pipeline para uma reunião aberta.

◆ Regra do projeto: nunca instanciar agentes diretamente — sempre via Orchestrator ou handle_rerun(). Esta página é somente leitura do hub — não dispara nenhum agente. Usar require_active_project() no topo.

06 Pitfalls aplicáveis a esta feature ▶
⚠ st.page_link() para app.py: nunca usar "app.py" em st.page_link — usar "pages/ValidationHub.py". O arquivo app.py não é uma página registrada.
⚠ Nested expander: Streamlit lança StreamlitAPIException se expander estiver dentro de outro expander. Esta página pode ter diagnósticos detalhados — usar st.caption() como header de seção ao invés de expanders aninhados.
⚠ href="#id" em components.html: se render_bpmn_diagnostics usar HTML com âncoras internas, as âncoras navegam o frame pai do Streamlit. Verificar a implementação em bpmn_diagnostics.py antes de reusar.
⚠ GitHub web editor: ValidationHub.py é um arquivo novo — criar via PyCharm/CLI, nunca pelo editor web do GitHub (risco de truncar conteúdo em edições futuras se o arquivo crescer).
